## Cross-Module Themes

### Recurring Skills Across All Exercises

Every exercise requires students to demonstrate:

1. **Architecture Design** (60 min)
   - Component diagrams
   - Technology selection with rationale
   - Design pattern application
   - NFR consideration (performance, security, cost, scalability)

2. **Analysis** (30 min)
   - Current state assessment
   - Gap analysis
   - Risk identification
   - Improvement opportunities

3. **Implementation Planning** (30 min)
   - Phased approach (3-5 phases)
   - Timeline estimation
   - Dependency mapping
   - Risk mitigation

4. **Stakeholder Communication** (30 min)
   - Executive summary (1-2 slides)
   - Visual architecture representation
   - Business value and ROI
   - Risk communication

### Standard Deliverables

All 50 exercises require identical deliverables:
- `analysis.md` (1-2 pages)
- `architecture.md` (2-3 pages)
- `implementation-plan.md` (1 page)
- `presentation.pdf` (5-7 slides)
- `diagrams/` directory

### Evaluation Criteria

Consistent across all exercises:
- **Architecture quality** (40%): Soundness, appropriate patterns
- **Documentation** (30%): Clarity, completeness, visual communication
- **Strategic thinking** (20%): Business alignment, long-term vision
- **Communication** (10%): Stakeholder-appropriate presentation

## Recommendations for Exercise Completion

### Template Completion Checklist

For each exercise, replace placeholders with:

1. **Specific Company Type**
   - Examples: "Fortune 500 financial services company", "Series B healthcare startup", "Global e-commerce platform"

2. **Concrete Requirements**
   - Technical: "Deploy 500+ ML models serving 10M daily predictions"
   - Business: "Reduce infrastructure costs by 30% while maintaining <100ms latency"

3. **Current Infrastructure Description**
   - Example: "On-premises Hadoop cluster (200 nodes), monolithic training pipeline, manual deployment process"

4. **Specific Pain Points**
   - Example: "Model deployment takes 2-3 weeks, no version control, limited observability, high GPU costs ($400K/month)"

5. **Clear Goals**
   - Example: "Implement automated MLOps platform supporting rapid experimentation, reduce deployment time to <1 day, achieve 99.9% uptime"

6. **Realistic Constraints**
   - Budget: "$2M initial investment, $500K annual operational budget"
   - Timeline: "6-month Phase 1 implementation, 12-month full rollout"
   - Resources: "Team of 8 engineers, existing AWS infrastructure, regulated industry (HIPAA)"

### Progressive Complexity Guidelines

Within each module, exercises should increase in complexity:

- **Exercise 1**: Foundation concepts, straightforward scenario
- **Exercise 2**: Intermediate complexity, additional constraints
- **Exercise 3**: Advanced considerations, trade-off evaluation
- **Exercise 4**: Enterprise-scale challenges, multi-stakeholder
- **Exercise 5**: Comprehensive scenario, integration of multiple concepts

### Scenario Realism

Scenarios should be:
- **Authentic**: Based on real-world AI infrastructure challenges
- **Specific**: Include concrete numbers, technologies, constraints
- **Relevant**: Aligned with current industry practices and tools
- **Challenging**: Require critical thinking and trade-off analysis
- **Actionable**: Provide sufficient context for students to develop solutions
